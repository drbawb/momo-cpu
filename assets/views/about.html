PANTHER150 Machine Emulator

The little CPU that could ...

  • Emulator
  • About

The PANTHER150 emulator is written in the Rust programming language. The machine is laid out as:

  • A single byte for the instruction pointer
  • Two bytes for the current instruction
  • An array of 16 bytes for the CPU registers.
  • An array of 256 bytes for the system memory.

The CPU matches the one described in class. The only notable difference is that instructions are numbered starting from 0, not 1.

Instructions

OP INSTRUCTION DESCRIPTION
0x0 MLOAD Load (Indirect)
0x1 RSET Load (Direct)
0x2 MSTOR Store (Indirect)
0x3 RMOV Register Move
0x4 ADDB Adds two bytes using 2s complement addition.
0x5 ADDF (Unimplmented!)
0x6 OR Logically ORs two bytes
0x7 AND Logically ANDs two bytes
0x8 XOR Logically XORs two bytes
0x9 ROT Rotates bits off the right edge to the left edge.
0xA JMPEQ Sets IP based on register 0 comparison.
0xB HLT Causes the machine to stop execution.
0xC-0xF RESERVED
