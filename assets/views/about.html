PANTHER150 Machine Emulator

The little CPU that could ...

  • About

The PANTHER150 emulator is written in the Rust programming language. The machine is laid out as:

  • A single byte for the instruction pointer
  • Two bytes for the current instruction
  • An array of 16 bytes for the CPU registers.
  • An array of 256 bytes for the system memory.

The CPU has a fetch, decode, execute cycle which decides to enter one of two states. The CPU can either continue with the Running state or it can enter the Halted state. The CPU will Halt if it encounters instruction 12 (0xB000) or if the CPU panics. The CPU might panic, for instance, if you were to use the unimplemented floating point arithmetic: or an undefined instruction.

The Run button will stop executing once the CPU halts. However you can force the machine to continue execution by hitting Run again. The Tick button will ignore the CPU state and fetch the next instruction regardless.
